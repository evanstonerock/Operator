Build a clean modern web app called Operator.

Purpose:
Operator is a personal life operating system for post-grad life. The app should help the user understand why a day was productive or unproductive, what to do tomorrow, and how to plan the week better using personal data.

Main concept:
This app combines two AI roles:
- Claude = deeper reflection, pattern analysis, honest explanation
- OpenAI = structured planning, practical next steps, action-oriented output

Important product rule:
Do not make this app bloated. Keep the architecture clean, modular, and understandable. Prioritize a working MVP over extra features.

Tech stack:
- Next.js
- React
- TypeScript
- Tailwind CSS
- Node.js API routes
- Clean modular file structure
- Ready to run on Replit
- Store secrets only in environment variables

Environment variables:
- OPENAI_API_KEY
- ANTHROPIC_API_KEY

Pages:
1. Daily Check-In
2. Weekly Review
3. History

Navigation:
- Simple top navigation or sidebar
- Easy switching between pages
- Minimal dashboard feel

================================
1. DAILY CHECK-IN PAGE
================================

Create a form with these fields:
- Notes about the day (large text area)
- Energy level (1–10)
- Focus level (1–10)
- Health / sickness level (1–10)
- Sleep quality (1–10)
- Mood (1–10)
- Tasks completed (text area or list input)
- Tasks missed (text area or list input)
- Optional habits completed
- Optional symptoms or sickness notes

Submit behavior:
When the form is submitted:
1. Send the daily check-in data to Claude first
2. Claude should return a reflective analysis that explains:
   - why the day was productive or unproductive
   - likely causes, patterns, or behavior issues
   - what felt off or what went right
   - one key insight from the day
3. Then send the original user data plus Claude’s reflection to OpenAI
4. OpenAI should return a structured plan for tomorrow with:
   - top priority
   - 3 key actions
   - suggested time-block plan
   - one thing to avoid
   - short summary of the ideal tone or goal for tomorrow

Output sections on the page:
- Claude Reflection
- OpenAI Tomorrow Plan
- Best Combined Advice

Best Combined Advice:
Create a short merged summary that combines Claude’s insight with OpenAI’s practical planning into one useful final recommendation.

Daily output rules:
- Keep advice specific to the entered data
- Do not give generic motivational fluff
- Keep output readable in a dashboard
- Claude should feel insightful and honest
- OpenAI should feel clear and action-oriented

================================
2. WEEKLY REVIEW PAGE
================================

Create a weekly review form with these fields:
- Weekly notes
- Main wins
- Main frustrations
- Energy trend
- Health trend
- Goals for next week
- Existing commitments

Weekly workflow:
1. Send the weekly data to Claude
2. Claude should return:
   - deeper weekly reflection
   - patterns across the week
   - likely reasons the week felt good, bad, or inconsistent
   - what underlying issue may be affecting productivity or recovery
3. Send the weekly data plus Claude’s reflection to OpenAI
4. OpenAI should return a structured next-week plan including:
   - focus days
   - admin days
   - recovery days
   - weekly priorities
   - risks to avoid

Weekly output sections:
- Claude Weekly Reflection
- OpenAI Weekly Plan
- Combined Weekly Advice

================================
